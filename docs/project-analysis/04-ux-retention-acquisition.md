# ScholarAIO 用户体验改进分析

## 背景

本报告聚焦两个目标：

1. 提升老用户留存，让已有用户更容易形成持续使用习惯
2. 提升新用户转化，让第一次接触项目的人更快感受到价值

本次分析基于当前代码库中的 README、docs、CLI、setup 流程、插件包装、workspace 和 insights 能力。

## 总体判断

ScholarAIO 的核心能力已经很强，覆盖了文献入库、检索、阅读、聚类、图谱、写作、导出和多 agent 集成。问题不在于功能缺失，而在于：

- 新用户第一次成功体验的路径还不够短
- 项目对外承诺与实际实现存在信息漂移
- 很多能促进留存的能力已经存在，但尚未被产品化地放大

当前项目更像“能力很强的研究引擎”，还没有完全打磨成“新用户一小时内就离不开、老用户每周都回来”的产品。

## 关键问题

### 1. 首次上手路径偏重

当前 Quick Start 主要覆盖安装、配置和启动 agent，但没有明确带用户完成一次完整的价值闭环，例如：

- 导入一篇论文
- 搜索一个主题
- 阅读摘要或结论
- 创建一个 workspace

结果是：用户知道项目“很强”，但不一定能在前 10 分钟内真的感受到它强在哪里。

### 2. 文档与元数据存在信息漂移

项目不同入口中的信息不完全一致，例如 skills 数量、MCP tools 数量、产品定位等。对于新用户来说，这会直接影响信任感，也会让老用户怀疑文档是否仍然可信。

### 3. 高频命令的人机反馈还不够产品化

CLI 主体能力已经齐全，但高频命令在以下方面还有提升空间：

- 中英文输出混杂
- 结果出来后没有把“下一步”明确告诉用户
- 某些错误提示仍偏向开发者视角，而不是用户视角

这会让项目更像“工具集合”，而不是“顺手的研究工作台”。

### 4. 留存能力存在，但没有形成主动闭环

当前已经有很好的留存基础：

- `notes.md` 跨会话分析记忆
- metrics 事件采集
- `insights` 阅读行为分析
- workspace 论文子集管理

但这些更像“存在的功能”，还不是“驱动用户回访的机制”。如果没有主动提醒、默认入口或下一步建议，用户不会自然养成周期性回来的习惯。

### 5. workspace 还偏“容器”，不够“项目驾驶舱”

workspace 现在能装论文、搜索、导出，但还没有把以下高价值信息直接呈现出来：

- 最近新增了什么
- 还有哪些论文没读
- 哪些论文还没形成笔记
- 这个主题接下来最值得做的 3 件事

如果 workspace 能成为项目入口，留存会明显增强。

## 优先级建议

### P0：高收益、低风险、应立即处理

1. 修正文档、插件元数据和对外描述中的信息漂移
2. 修 setup 中会误导用户的错误命令示例
3. 统一高频 CLI 路径中的文案语言和语气
4. 在搜索、阅读、索引等高频命令后补充下一步提示

这类改动实现成本低，但会显著提升第一印象、降低新手困惑，并提升日常使用的顺滑度。

### P1：最影响留存的产品化增强

1. 增加真正的 onboarding 流程
2. 让 workspace 成为长期入口
3. 强化 `insights` 的主动价值

建议方向：

- 新增 `scholaraio start` 或首启引导
- 新建 workspace 后自动提示下一步动作
- 定期输出研究行为摘要、未读邻近论文、可继续推进的主题

### P2：最影响拉新的展示增强

1. 补 logo、demo GIF、真实使用截图
2. 增加“3 个典型场景”而不是只列功能
3. 提供示例数据或 sample workspace，缩短首次体验路径

## 推荐立刻执行的动作

### 动作一：统一产品信息

目标：

- 所有公开入口对“项目是什么、支持多少能力、推荐怎么用”给出一致说法

收益：

- 直接提升新用户信任感
- 降低“文档是不是过期了”的心理成本

### 动作二：优化首批高频 CLI 反馈

目标：

- 搜索、阅读、索引、向量化等操作都使用统一语言风格
- 结果出来后明确告诉用户下一步怎么做

收益：

- 提高第一次成功率
- 降低“我接下来该干嘛”的中断
- 让 CLI 更像产品而不是命令集合

### 动作三：把留存功能前置到用户主路径

目标：

- 让 `workspace` 和 `insights` 成为老用户的日常入口

收益：

- 增强复访理由
- 把“偶尔查询”升级成“持续研究工作流”

## 建议跟踪的指标

如果后续要持续优化用户体验，建议追踪以下指标：

1. 首次激活时间
   从安装完成到第一次成功搜索 / 阅读 / 创建 workspace 的耗时

2. 首次成功率
   新用户是否能在第一小时内完成至少一个完整闭环

3. workspace 使用率
   有多少用户在检索后继续创建或使用 workspace

4. 周活跃研究行为
   每周是否持续有 search、show、ws、export、writing 等行为

5. 回访触发来源
   用户回来的主要入口是搜索、workspace、insights 还是写作

## 结论

ScholarAIO 当前最缺的不是新功能，而是产品化包装：

- 更短的上手路径
- 更一致的对外信息
- 更明确的下一步引导
- 更主动的留存触发机制

如果只选三件事优先做，建议顺序是：

1. 修信息漂移和错误指引
2. 优化高频 CLI 反馈与下一步提示
3. 把 workspace 和 insights 做成真正的长期入口
